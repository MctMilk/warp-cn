# i18n 贡献者指南

本 fork 的客户端 UI 已全量本地化为中文（默认）+ 英文（fallback）。本指南面向**新增 / 修改 UI 字符串**的贡献者。

## 架构速览

| 组件 | 路径 | 作用 |
|------|------|------|
| 运行时 crate | `crates/warp_i18n/` | bundle 加载、locale 切换、`tr!` 宏 |
| Proc-macro | `crates/warp_i18n_macros/` | `t!` 编译期校验 key 存在 |
| 翻译数据 | `crates/warp_i18n/bundles/{en,zh-CN}/*.ftl` | Fluent 文件，按 namespace 拆 |
| Lint | `xtask check-i18n` | AST 扫描 UI 构造点 |
| Allowlist | `crates/warp_i18n/lint_allowlist.toml` | 豁免清单（CLI / DSL token / etc.） |

## 新增 UI 字符串的工作流

```rust
use warp_i18n::t;

let label = Text::new(t!("namespace-key"), font, size);
let with_arg = Text::new(t!("greeting-name", name = user.name()), font, size);
```

1. **选 namespace**：`menu` / `settings_*` / `terminal` / `dialog` / `notification` / `onboarding` / `ai_ui` / `coding` / `misc` 等。Key 必须以对应前缀开头（如 `terminal-zero-new-session`）。
2. **添加 entry**：在 `bundles/en/<ns>.ftl` 与 `bundles/zh-CN/<ns>.ftl` 同步加 entry。两边 key 集合必须严格一致（否则 `--check-parity` 失败）。
3. **写代码**：用 `t!("namespace-key")`。`t!` 不接受非字面量 key —— 动态 key 必须用 `tr!`，且后者运行时缺 key 会 fallback 至 en。
4. **校验**：`cargo check -p warp_i18n` —— `build.rs` 解析全部 .ftl 校验语法 + 生成 key index；`t!` proc-macro 与该索引比对，未命中 key 直接 `compile_error!`。

## Lint 工作流

```bash
# 仅看 warning（开发期）
cargo xtask check-i18n --mode warning

# CI 同款（任意 warning 即失败）
cargo xtask check-i18n --mode hard

# 校验 en/zh-CN bundle 对齐
cargo xtask check-i18n --check-parity
```

Lint 识别 UI 构造点（`Text::new`、`Button::new`、`.label`/`.tooltip`/`.title`/`.placeholder`/`.description`、`Menu::new` 等）的字符串字面量，命中即报告。豁免规则：

- `tracing!` / `log!` / `println!` / `panic!` / `assert!` 等开发者面宏
- `#[cfg(test)]` 与 `*_test.rs` / `*_tests.rs` / `**/examples/**`
- `crates/warp_cli/**` 与 `crates/warp_cli_*/**`（CLI 不汉化，design.md D14）
- `lint_allowlist.toml` 显式登记的字面量

## Allowlist 增长规则

`lint_allowlist.toml` 在 Phase 5 固化，`phase5_baseline_count = 4`。CI 在 PR 中对比 head 与 base 的 entries 数：

- 不增长 → 通过
- 增长且 PR 带 `i18n-allowlist-grow` label → 通过（review 责任）
- 增长但无 label → CI 拒绝

新增条目 schema（design.md D21）：

```toml
[[entries]]
file_glob = "<glob>"               # globset 语法
callsite = "<path::to::fn>"        # 精确路径匹配
literal = "<exact source value>"   # 字符串字面量值
reason = "<≥10 字符的说明>"
added_phase = 5
owner = "<owner-name>"
```

## 翻译流程

- 翻译产出由 AI agent 按 namespace 一次性产出，参照 `crates/warp_i18n/GLOSSARY.md` 锁定术语。
- 修订英文文案：编辑 `bundles/en/*.ftl` → 更新对应 `bundles/zh-CN/*.ftl` → 跑 `--check-parity` → PR。
- 与上游 merge 冲突：见 `crates/warp_i18n/MERGE_NOTES.md` 的两步 split 策略。

## Locale 解析与切换

设置项 `language` 写入 `~/.warp/settings.toml`：

| 值 | 行为 |
|------|------|
| `"zh-CN"` / `"en"` | 锁定该语言 |
| `"system"` | 跟随系统 locale，`zh*` → ZhCn，否则 → En |
| 缺失（默认） | `Locale::ZhCn`（fork 默认） |

切换无需重启进程：`warp_i18n::set_locale()` 通过 `arc-swap::ArcSwap<Locale>` 发布新值，`tokio::sync::watch` 通知 UI 层重渲染。`t!()` lock-free 读路径，P99 < 10μs。

## 性能 & 测试

- 微基准：`cargo bench -p warp_i18n` 验证 lookup P99
- PBT：`crates/warp_i18n/tests/property_*.rs` 覆盖渲染纯函数性、并发、locale 解析等不变量

## 常见问题

**Q：Lint 把 `tracing::info!("foo")` 误报了？**  
A：豁免规则按 callsite 路径精确匹配。检查 `xtask/src/i18n_lint/sites.rs` 的 `call_sites` / `method_sites` 是否误把日志宏纳入。

**Q：服务端拉取的字符串需不需要 `t!`？**  
A：不需要。把字段类型改为 `RemoteString`（`crates/warp_server_client::RemoteString`），lint 会硬阻断对其调用 `t!`/`tr!`。

**Q：键位提示要不要本地化？**  
A：不要。macOS 渲染符号 `⌘⇧⌥⌃`，Win/Linux 渲染英文 `Ctrl/Shift/Alt`。修饰键名（`Cmd`/`Ctrl`/`Shift`/`Alt`/`Option`/`Meta`）已进 allowlist。
